Understanding Relative & Absolute Positioning

This task demonstrates how relative and absolute positioning work together. The notification badges are absolutely positioned within their card containers.

Key Concepts:

1. position: relative (Parent)

.card { position: relative; /* Creates a positioning context for absolutely positioned children */ }

Applying position: relative to the parent card doesn't move it from its normal position, but it creates a positioning context. Any absolutely positioned children will be positioned relative to this parent instead of the document body.

2. position: absolute (Child)

.notification-badge { position: absolute; top: -12px; right: -12px; /* Positioned relative to the nearest positioned ancestor (the card) */ }

The badge uses top: -12px and right: -12px to position it at the corner. Because the parent has position: relative, the badge is positioned relative to the card edges.

  • Relative positioning is used on parent elements to create a positioning context
  • Absolute positioning removes elements from the document flow
  • Absolutely positioned elements are positioned relative to their nearest positioned ancestor
  • If no positioned ancestor exists, they're positioned relative to the document body
  • This pattern is perfect for badges, tooltips, popovers, and dropdown menus
  • Always ensure the parent has a position value (relative, absolute, fixed, sticky, or sticky)

Real-world applications: UI badges, dropdown menus, tooltips, notification counters, custom cursors
