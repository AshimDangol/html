Section 1: Understanding Fixed Positioning

The header and footer above and below are using position: fixed. This removes them from the normal document flow and keeps them visible even when scrolling. The main content has margin-top and margin-bottom to prevent overlap with fixed elements.

Section 2: How Fixed Positioning Works

Fixed elements are positioned relative to the viewport (the visible area of the browser). They stay in the same position even when the page is scrolled. Common use cases include navigation bars, chat widgets, and floating action buttons.

Section 3: Scroll Down to See the Effect

Notice how the header remains at the top and the footer at the bottom as you scroll. The z-index property ensures they appear above other content. This is a fundamental technique used in modern web design.

Section 4: More Content

The z-index property is crucial for fixed elements. It determines the stacking order of elements on the page. Higher z-index values appear on top of lower values.

Section 5: Real-World Applications

Fixed positioning is used in many real-world scenarios:

  • Navigation bars that stay visible while scrolling
  • Chat windows in the bottom-right corner
  • Cookie consent banners
  • Back-to-top buttons

Section 6: Viewport vs Document Flow

Unlike relative or absolute positioning, fixed elements are positioned relative to the viewport, not their parent element. This makes them perfect for persistent UI elements.

Section 7: Continue Scrolling

Keep scrolling to experience how the fixed header and footer remain in place while the content moves. This is a key concept in understanding CSS positioning.

Section 8: End of Content
