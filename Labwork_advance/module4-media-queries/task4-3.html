Using display: none and display: block with Media Queries

The most common way to hide and show content is using the display property with media queries. The sidebar uses display: none on mobile and the mobile message uses display: none on desktop.

/* Desktop and Tablet: Sidebar visible */ .sidebar { display: block; /* Default - shown */ } .mobile-message { display: none; /* Hidden on desktop */ } /* Mobile: Sidebar hidden, message visible */ @media (max-width: 768px) { .sidebar { display: none; /* Hidden on mobile */ } .mobile-message { display: block; /* Shown on mobile */ } .main-layout { grid-template-columns: 1fr; /* Single column */ } }

Advanced Hiding Techniques

Besides display: none, there are other ways to hide content:

1. visibility: hidden

Hides the element but still takes up space in the layout.

.element { visibility: hidden; /* Hides but reserves space */ } .element { visibility: visible; /* Shows */ }

2. opacity: 0

Makes element transparent but it still occupies space and can be interacted with.

.element { opacity: 0; /* Invisible but interactive */ pointer-events: none; /* Prevent interaction */ }

3. max-height: 0

Collapses the element to zero height. Great for smooth animations.

.element { max-height: 0; overflow: hidden; transition: max-height 0.3s ease; } .element.open { max-height: 500px; }

4. position: absolute + left: -9999px

Moves element off-screen. Used for accessibility - screen readers can still read it.

.sr-only { position: absolute; left: -9999px; width: 1px; height: 1px; overflow: hidden; }

Best Practices for Hiding Content

‚úÖ Use display: none for

  • Content that shouldn't be rendered at all on certain screens
  • Alternative layouts for different devices
  • Hamburger menus (instead of full navigation)
  • Desktop-specific sidebars

‚ùå Avoid

  • Hiding important content on mobile (makes site unusable)
  • Using visibility: hidden in layouts (wastes space)
  • Relying on z-index to hide overlapping content
  • Loading content you plan to hide (wastes bandwidth)

üí° Pro Tips

  • Test on real devices, not just browser emulation
  • Consider mobile-first: show by default, hide if needed
  • Use semantic HTML elements for accessibility
  • Provide alternative navigation on mobile
  • Keep important information visible on all devices

Real-World Examples

  • Admin Dashboards: Hide sidebar on mobile, show hamburger menu
  • E-commerce: Hide product filters on mobile, show via dropdown
  • Blogs: Hide sidebar on mobile to focus on content
  • News Sites: Different layouts for mobile, tablet, desktop
  • Social Media: Hide detailed previews on mobile, show text only
  • Forms: Show multiple columns on desktop, single column on mobile
