Understanding Responsive Navigation

This page demonstrates how navigation changes based on screen size using CSS media queries. Try resizing your browser window to see the transformation:

  • Desktop (wider than 768px): Navigation displays horizontally
  • Mobile (768px and below): Navigation becomes a hamburger menu

Desktop Layout (Default)

On desktop screens, the navigation menu is displayed horizontally next to the logo. This provides quick access to all menu items without extra clicks.

/* Desktop Navigation - Horizontal Layout */ nav ul { display: flex; list-style: none; gap: 5px; } nav a { padding: 12px 20px; border-bottom: 3px solid transparent; } nav a:hover { border-bottom-color: #ffd700; }

Mobile Layout (Media Query)

On mobile screens (768px and below), the navigation transforms into a toggleable menu accessed via a hamburger button (â˜°). This saves space and improves mobile usability.

@media (max-width: 768px) { /* Show hamburger button */ .hamburger { display: block; } /* Hide nav by default, show when active */ nav { display: none; } nav.active { display: flex; } /* Stack menu items vertically */ nav ul { flex-direction: column; } /* Adjust styling for mobile */ nav a { padding: 15px 20px; border-left: 3px solid transparent; } }

How the Hamburger Menu Works

The hamburger button toggles a CSS class on the navigation menu. When you click the button, JavaScript adds or removes the "active" class, which changes the display property.

/* JavaScript for hamburger menu */ const hamburger = document.getElementById('hamburger'); const navMenu = document.getElementById('nav-menu'); hamburger.addEventListener('click', () => { navMenu.classList.toggle('active'); hamburger.classList.toggle('active'); }); /* CSS for toggled state */ nav.active { display: flex; }

Key Breakpoints Explained

Mobile First Approach: Start with mobile styles, then use media queries to enhance for larger screens.
  • Mobile: max-width: 480px - Small phones
  • Tablet: 481px - 768px - Tablets and large phones
  • Small Desktop: 769px - 1024px - Small laptop screens
  • Large Desktop: 1025px+ - Full-size monitors

Real-World Applications

This responsive navigation pattern is used by virtually all modern websites:

  • News websites and blogs
  • E-commerce platforms (Amazon, eBay, etc.)
  • Social media sites (Facebook, Twitter, Instagram)
  • SaaS applications
  • Portfolio and personal websites
  • Corporate websites

Try It Out!

Resize your browser window to trigger the media query breakpoint at 768px. You'll see:

  • Navigation menu disappears above the breakpoint
  • Hamburger button appears below the breakpoint
  • Menu items stack vertically when shown on mobile
  • Click the hamburger button to toggle the menu
