Creating Responsive Grids with Media Queries

This dashboard demonstrates how CSS Grid and media queries work together to create layouts that adapt to different screen sizes. The number of columns automatically changes based on the viewport width.

Responsive Grid CSS:

/* Desktop: 4 columns */ .stats-grid { display: grid; grid-template-columns: repeat(4, 1fr); gap: 20px; } /* Tablet: 2 columns */ @media (max-width: 1024px) { .stats-grid { grid-template-columns: repeat(2, 1fr); } } /* Mobile: 1 column */ @media (max-width: 640px) { .stats-grid { grid-template-columns: 1fr; } }

Breakpoint Strategy:

Device Type Screen Width Columns Best For
Mobile Phone Up to 640px 1 column iPhone, Android phones
Tablet 641px - 1024px 2 columns iPad, tablets
Desktop 1025px+ 4 columns Laptops, desktops, wide screens

Common Breakpoints:

Standard Industry Breakpoints:
  • Mobile: 320px - 480px (small phones)
  • Mobile: 480px - 768px (phones to tablets)
  • Tablet: 768px - 1024px
  • Desktop: 1024px - 1440px
  • Large Desktop: 1440px+ (4K screens)

Mobile-First vs Desktop-First:

Mobile-First Approach (Recommended):

/* Start with mobile layout */ .stats-grid { display: grid; grid-template-columns: 1fr; /* 1 column for mobile */ } /* Add features for larger screens */ @media (min-width: 641px) { .stats-grid { grid-template-columns: repeat(2, 1fr); } } @media (min-width: 1025px) { .stats-grid { grid-template-columns: repeat(4, 1fr); } }

Desktop-First Approach (Less Common):

/* Start with desktop layout */ .stats-grid { grid-template-columns: repeat(4, 1fr); } /* Reduce for smaller screens */ @media (max-width: 1024px) { .stats-grid { grid-template-columns: repeat(2, 1fr); } } @media (max-width: 640px) { .stats-grid { grid-template-columns: 1fr; } }

Try It Out:

Resize your browser window to see the dashboard transform:

  • Wide Desktop (1025px+): 4 cards per row
  • Tablet (641px - 1024px): 2 cards per row
  • Mobile (up to 640px): 1 card per row (stack vertically)
